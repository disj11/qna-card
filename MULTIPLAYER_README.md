# 🌐 원거리 함께하기 (멀티플레이어) 가이드

서로 다른 폰에서 P2P(WebRTC)로 연결해 실시간으로 질문카드를 함께 진행하는 모드입니다.

## 🌟 주요 기능

### 1. P2P 연결
- PeerJS를 사용한 브라우저 간 직접 연결
- 게임 데이터는 서버를 거치지 않고 참가자 간에 직접 전송됩니다
- STUN 서버를 통한 NAT 통과

### 2. 실시간 채팅 & 이모지 반응
- 게임 중 자유로운 대화, 참가자 입장/퇴장 시스템 메시지
- 화면에 떠오르는 애니메이션과 함께 표시되는 이모지 반응

### 3. 턴 기반 진행
- 현재 차례인 참가자가 카드를 넘기며 진행
- 레벨(워밍업/연결/진심)과 진행 상황이 양쪽 화면에 동기화됨

## 🚀 시작하기

### 방 만들기 (호스트)

1. 메뉴에서 **"원거리 함께하기"** 선택
2. **"방 만들기"** → 닉네임 입력
3. 자동 생성된 **방 코드**(예: `ABC123`)를 상대방에게 공유
4. 상대방이 입장할 때까지 대기실에서 기다리기
5. 모두 "준비 완료" 상태가 되면 **"게임 시작"**

### 방 참가하기

1. 메뉴에서 **"원거리 함께하기"** 선택
2. **"방 참가하기"** → 닉네임과 방 코드 입력 (대소문자 구분 없음)
3. **"준비 완료"** 클릭 후 호스트가 게임을 시작할 때까지 대기

## 💬 채팅 사용하기

- 화면 우하단 채팅 아이콘을 눌러 메시지를 주고받을 수 있습니다
- 이모지 버튼(😊)으로 빠른 반응을 남길 수 있습니다
- 채팅창 헤더를 눌러 최소화/펼치기 가능

## 🔧 기술 스택

- **Frontend**: React 19 + TypeScript + Tailwind CSS + Vite
- **P2P 통신**: PeerJS(WebRTC 래퍼) + STUN 서버

관련 코드 위치:
- `src/features/multiplayer/lib/P2PConnection.ts` — P2P 연결 관리
- `src/features/multiplayer/lib/peerConfig.ts` — PeerJS 호스트/STUN 설정
- `src/features/multiplayer/hooks/` — 방/턴/채팅 상태 훅
- `src/features/multiplayer/MultiplayerPage.tsx` — 화면 진입점

## 🔒 개인정보 보호

- **P2P 방식**: 모든 게임 데이터는 참가자 간 직접 전송됩니다
- **서버 미저장**: 채팅, 답변 등 어떤 데이터도 서버에 저장되지 않습니다
- **익명성**: 입력한 닉네임 외에 개인정보는 수집하지 않습니다
- **세션 종료**: 브라우저를 닫으면 모든 데이터가 삭제됩니다

## 🐛 문제 해결

연결이 안 되거나 "생성 중..."에서 멈추는 경우 [로컬 PeerJS 서버 설정 & 문제 해결 가이드](LOCAL_PEERJS_SETUP.md)를 참고하세요. 멀티플레이어가 작동하지 않아도 **"한 기기로 함께"** 또는 **"혼자 미리보기"** 모드는 네트워크 연결 없이 정상적으로 사용할 수 있습니다.
